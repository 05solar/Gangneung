# 강릉 여행 일정 · 프론트엔드 (Frontend)

강릉 여행 일정 앱의 프론트엔드입니다. **React + Vite + TypeScript(TSX)** 로 구현했으며,
디자인은 `stitch_y2k_mobile_itinerary` 의 **"Ethereal Y2K"** 감성(소프트-레트로 미니멀리즘)을 따릅니다.
백엔드 Express API 에서 일정 데이터를 받아 화면을 그립니다.

---

> CSS 변수(`:root`)로 한곳에 모아 관리합니다.

## 🚀 실행 방법

> 백엔드(`../backend`)를 먼저 실행해야 데이터가 표시됩니다.

```bash
# 1) 백엔드 실행 (다른 터미널)
cd backend && npm install && npm start   # http://localhost:4000

# 2) 프론트엔드 실행
cd frontend
npm install
npm run dev        # http://localhost:5173
```

개발 서버의 `/api` 요청은 `vite.config.ts` 프록시를 통해 백엔드(4000)로 전달됩니다.

빌드:
```bash
npm run build      # dist/ 에 정적 파일 생성
npm run preview    # 빌드 결과 미리보기
```

도커(루트에서 프론트+백엔드 통합 실행):
```bash
cd ..              # 프로젝트 루트
docker compose up --build    # http://localhost:8080
```

## 🧭 화면 구성

| 경로 | 페이지 | 내용 |
|------|--------|------|
| `/day/1` | 일정 · Day 1 | 강릉시외터미널 도착 → 빵다방 → 소품샵 → 카페 꾸옥 → 샌마르(양식) → 누그나스(술집) → 숙소 체크인 |
| `/day/2` | 일정 · Day 2 | 초당순두부 → 순두부젤라또 → 커피거리 → 길감자 → 감자유원지 → 아라나비 → 해수욕장 → 숙소 샤워 → 주문진 저녁 |
| `/day/3` | 일정 · Day 3 | 소돌 바다산책 → 주문진 수산시장 → 브런치/카페 → 터미널 귀가 |
| `/game` | 게임 홈 | 플레이어 6명 선택 → 게임 선택 + 총합 랭킹 |
| `/game/mine` | 돌깨기 | 돌 터치 +10점, 정산으로 점수 획득 |
| `/game/watermelon` | 수박게임 | 같은 과일을 합쳐 수박 만들기 |
| `/game/2048` | 2048 | 타일을 밀어 합치기 |
| `/game/wordle` | 워들 | 5글자 영어 단어 맞히기 |
| `/game/blackjack` | 블랙잭 | 보유 점수로 베팅하는 카드 게임 |
| `/game/omok` | 오목 | 규칙기반 AI와 15×15 대국 |

- 하단 탭은 **일정 / 게임** 두 개이며, 일정 안에서는 폴더 탭으로 Day를 전환합니다.
- 각 일정 카드는 **우측에 작은 이미지 박스**, **이동수단 배지(택시/도보)** 를 표시합니다.
- 일정 사이의 **이동수단 미니박스**에는 **택시/도보 · 약 N분**(구간 소요시간)이 표시되며,
  누르면 해당 구간(출발→도착)이 입력된 **카카오맵 길찾기**가 새 탭으로 열립니다.
  - 소요시간은 각 장소의 **실제 좌표 기반 도로거리**로 계산한 추정값입니다.
- **1일차**는 터미널 도착 후 **주문진 숙소에 얼리 체크인하여 짐을 먼저 보관**한 뒤 시내 일정을 진행합니다.

## 🎮 게임

- **플레이어**: 6명 중 한 명을 선택하며, 점수는 **플레이어별로 백엔드에 누적 저장**됩니다.
- **누적 점수 · 총합 랭킹**: 각 게임을 **끝낼 때마다 그 판의 점수가 보유 점수에 누적**되고,
  모든 게임 점수를 합산한 **총합 랭킹**이 게임 홈 하단에 표시됩니다. **모두 0점에서 시작**합니다.
- 게임 목록의 각 항목에는 `src/lib/gameArt.ts` 로 **생성한 게임 이미지**가 붙습니다.
- 5가지 게임:
  - **돌깨기** — 곡괭이로 **돌을 터치할 때마다 +10점**. **정산** 버튼을 누르면 캔 점수가 바로 보유
    점수에 반영됩니다. 돌·곡괭이 이미지는 `gameArt.ts` 의 SVG 로 생성. (점수를 모으는 기본 수단)
  - **수박게임** — `matter-js` 물리. **과일이 빨간 경고선을 넘는 순간 즉시 게임 오버**되어 그때
    점수가 누적됩니다. 과일은 `fruitSprites.ts` 의 SVG 캐릭터로 그려집니다.
  - **2048** — 방향키/스와이프로 타일 합치기. 타일이 **transform 으로 미끄러지는 슬라이딩
    애니메이션**과 합체 팝 효과가 있고, 칸 크기는 고정입니다.
  - **워들** — 6번 안에 5글자 영어 단어 맞히기. 적은 시도로 맞힐수록 높은 점수(600~100).
  - **블랙잭** — **설명창** 제공. **보유 점수에서 베팅**하여 대결, 승패에 따라 점수가 오르내립니다.
    카드는 **왼쪽에서 플립하며 나타나는 애니메이션**(딜·히트). **보유 점수가 없으면 플레이 불가**(잠금).
  - **오목** — 15×15 판에서 **규칙기반 AI**와 대결. 흑돌로 먼저 5개를 연결하면 승리(+120점).
    AI는 각 지점의 공격/수비 패턴 점수(열린3·4 등)를 계산해 위협을 막고 자신의 연결을 늘립니다.
- 게임 화면은 `/game/<id>?p=<플레이어id>` 로 **딥링크** 접속도 가능합니다.

## 📱 일정 화면 (모바일 최적화)

- 일정 카드는 **컴팩트 가로형**(작은 썸네일 + 시간·제목 / 장소·이동수단 2줄)으로,
  **360×740 화면에서 한 번에 4개 이상**의 일정이 보이도록 높이를 최소화했습니다.
